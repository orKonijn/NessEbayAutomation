# ReadMeAIBugs

## סקירת בעיות בקוד

בבדיקה סטטית של הקוד נמצאו מספר בעיות שעלולות לגרום לבדיקה להיות לא יציבה, לא יעילה, או לא באמת לבדוק את ההתנהגות הרצויה.

---

## 1. שימוש ב־`time.sleep` במקום פונקציות ש־Playwright מנגיש

הקוד משתמש ב־`time.sleep`, שעוצר את הבדיקה לזמן קבוע.

זו בעיה מכיוון שלא תמיד אלמנטים ואתרים נטענים באותה מהירות. אם האתר איטי יותר מהזמן שהוגדר, הבדיקה עלולה להיכשל. מצד שני, אם האתר או האלמנט כבר נטענו, הבדיקה סתם מחכה ומבזבזת זמן ריצה.

בפרויקט עם הרבה בדיקות, שימוש כזה יכול להאט משמעותית את כל ההרצה.

מניסיון אישי, מומלץ להשתמש בפונקציות ההמתנה של Playwright, משום שהן מייעלות את זמן הריצה וגורמות לפחות בעיות יציבות.

### תיקון מומלץ

במקום להשתמש ב־`time.sleep`, עדיף להשתמש ב־`expect` או בהמתנות מובנות של Playwright.

לדוגמה:

```python
from playwright.sync_api import expect

search_box = page.locator("#search")
expect(search_box).to_be_visible()
```

## 2. Import של Selenium

בקוד מופיע ה־import הבא:

```python
from selenium import webdriver
```

אבל בפועל אין שימוש בכלל ב־Selenium בקטע הקוד.

דבר זה עלול ליצור בלבול בין כלי האוטומציה, מכיוון שהבדיקה עצמה נכתבת עם Playwright. בנוסף, זה מוסיף תלות ב־Selenium ללא צורך ומקשה על ההבנה והתחזוקה של הקוד.

### תיקון מומלץ

למחוק את השורה:

```python
from selenium import webdriver
```

## 3. הסלקטורים כלליים ושבירים מדי

הקוד משתמש בסלקטורים כמו:

```python
#search
.button
.result-item
```

הבעיה היא שסלקטורים כאלה תלויים מאוד במבנה ובעיצוב של הדף. אם העיצוב של האתר משתנה, או אם מתווסף עוד כפתור עם אותו class, הבדיקה יכולה להישבר או ללחוץ על הכפתור הלא נכון.

בנוסף, הבדיקה תעבוד בצורה תקינה רק אם יש אלמנט אחד שמתאים לכל סלקטור בדף, דבר שלרוב אינו נכון בפרויקטים אמיתיים.

### תיקון מומלץ

מומלץ להשתמש בסלקטורים יציבים יותר, כמו `data-testid`.

לדוגמה:

```python
page.get_by_test_id("search-input").fill("playwright testing")
page.get_by_test_id("search-button").click()
```

אפשרות נוספת היא להשתמש ב־role כאשר יש שם נגיש ברור:

```python
page.get_by_role("button", name="Search").click()
```

## 4. אין `assert` או `expect`

הבדיקה עצמה לא באמת מאמתת תוצאה כלשהי, משום שהיא לא קוראת ל־`expect` או `assert`.

המשמעות היא שאם אין שגיאה טכנית בקוד עצמו, הבדיקה יכולה לעבור גם אם החיפוש לא באמת עבד, גם אם אין תוצאות, וגם אם התוצאה שהתקבלה אינה נכונה.

### תיקון מומלץ

להוסיף בדיקה שמוודאת שהתוצאה הרצויה התקבלה.

לדוגמה:

```python
from playwright.sync_api import expect

results = page.locator(".result-item")
expect(results.first).to_be_visible()
```

או אם רוצים לוודא שיש לפחות תוצאה אחת:

```python
assert results.count() > 0
```

עדיף להשתמש ב־`expect` של Playwright, כי הוא יודע לבצע המתנה אוטומטית עד שהתנאי מתקיים או עד שנגמר timeout.
